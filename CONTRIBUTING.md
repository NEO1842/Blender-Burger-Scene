# Contributing Guidelines / 貢献ガイドライン

---English---

Thank you for your interest in contributing to **Blender Burger Scene**! 🍔

## How to Contribute

### 🐛 Reporting Bugs
- Open an [Issue](https://github.com/NEO1842/Blender-Burger-Scene/issues)
- Describe what you expected vs. what actually happened
- Include your Blender version and OS if relevant

### 💡 Suggesting Improvements
- Open an [Issue](https://github.com/NEO1842/Blender-Burger-Scene/issues) with the label `enhancement`
- Explain your idea clearly and why it would be useful

### 🔧 Submitting Changes (Pull Requests)
1. Fork this repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with a clear message: `git commit -m "Add: description of change"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request and describe your changes

## Notes
- Please keep changes focused and minimal
- If redistributing modified files, include a link back to this repository (see LICENSE)

---日本語---

**Blender Burger Scene** に興味を持っていただきありがとうございます！🍔

## 貢献の方法

### 🐛 バグの報告
- [Issue](https://github.com/NEO1842/Blender-Burger-Scene/issues) を開いてください
- 期待した動作と実際の動作を説明してください
- 必要に応じてBlenderのバージョンとOSを記載してください

### 💡 改善の提案
- `enhancement` ラベルをつけて [Issue](https://github.com/NEO1842/Blender-Burger-Scene/issues) を開いてください
- アイデアと、それがなぜ有用かを説明してください

### 🔧 変更の提出（プルリクエスト）
1. このリポジトリをフォークする
2. 新しいブランチを作成：`git checkout -b feature/your-feature-name`
3. 変更を加える
4. 明確なメッセージでコミット：`git commit -m "Add: 変更内容の説明"`
5. フォークにプッシュ：`git push origin feature/your-feature-name`
6. プルリクエストを開き、変更内容を説明する

## 注意事項
- 変更はできるだけシンプルで的絞ったものにしてください
- 改変ファイルを再配布する場合は、このリポジトリへのリンクを含めてください（LICENSEを参照）
